## Why

The server is plain JavaScript (ESM) with a structural smell: `tasks.js`/`habits.js` exist both as Express routers at the project root and as same-named data-access modules in `lib/`, which is confusing to navigate. There is also no static typing, so request/response shapes, the `req.userId` field added by middleware, and validation enums (`VALID_STATUS`, `VALID_PRIORITY`, `VALID_FREQUENCY`) are unchecked and can drift from the Prisma schema. This change tidies the file layout and migrates the codebase to TypeScript to catch this class of bug at build time, with no change to the public HTTP API behavior.

## What Changes

- Move route files (`quotes.js`, `pictures.js`, `tasks.js`, `habits.js`) into a new `routes/` directory.
- Rename the `lib/` data-access modules that collided with route names: `lib/tasks.js` → `lib/tasks-repository.ts`, `lib/habits.js` → `lib/habits-repository.ts`.
- Convert all source files (`server.js`, everything under `routes/` and `lib/`) to TypeScript (`.ts`).
- Add `tsconfig.json` and TypeScript dev dependencies (`typescript`, `tsx`, `@types/node`, `@types/express`, `@types/cors`).
- Switch `dev`/`start` scripts to run via `tsx` (no separate build/compile step).
- Replace hand-maintained validation arrays (`VALID_STATUS`, `VALID_PRIORITY`, `VALID_FREQUENCY`) with the corresponding enums generated by Prisma Client (`TaskStatus`, `Priority`, `HabitFrequency`) as the single source of truth. Validation logic stays manual (if/else), not migrated to a schema library.
- Add an Express `Request` type augmentation for `req.userId` (set by `requireAuth` middleware).
- Update `vercel.json` to point at the new TypeScript entrypoint.
- **BREAKING** (internal only): import paths for route and repository modules change; no impact on the public HTTP API or its consumers (browser extension client).

## Capabilities

### New Capabilities
- `server-codebase`: structural and tooling requirements for the server source — TypeScript as the implementation language, route/repository file layout, and the dev/start/deploy toolchain.

### Modified Capabilities
(none — public HTTP request/response behavior for quotes, pictures, tasks, and habits is unchanged)

## Impact

- Affected files: `server.js`, `quotes.js`, `pictures.js`, `tasks.js`, `habits.js`, all of `lib/*.js`, `package.json`, `vercel.json`.
- New files: `tsconfig.json`, `routes/*.ts`, plus a `types/express.d.ts` (or similar) for the `req.userId` augmentation.
- Dependencies: adds `typescript`, `tsx`, `@types/node`, `@types/express`, `@types/cors` as devDependencies. No runtime dependency changes.
- Deployment: `vercel.json` build entrypoint changes from `server.js` to `server.ts`; `@vercel/node` transpiles TypeScript directly so no separate build step is introduced.
- No database schema changes, no API contract changes.
