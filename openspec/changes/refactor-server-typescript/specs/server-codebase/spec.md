## ADDED Requirements

### Requirement: Server source is TypeScript
The server SHALL be implemented in TypeScript. All Express route handlers, middleware, and shared utilities SHALL be `.ts` files type-checked by `tsc`.

#### Scenario: Type checking passes
- **WHEN** `tsc --noEmit` is run against the project
- **THEN** it completes with zero type errors

#### Scenario: Authenticated request has a typed userId
- **WHEN** a route handler runs after the `requireAuth` middleware
- **THEN** `req.userId` is typed as `string` (via an Express `Request` type augmentation) rather than `any`

### Requirement: Route and repository modules do not share filenames
Express router modules SHALL live under `routes/`, and data-access modules SHALL live under `lib/` with names that do not collide with any router module.

#### Scenario: No duplicate basenames across routes/ and lib/
- **WHEN** listing files in `routes/` and `lib/`
- **THEN** no filename (ignoring extension) appears in both directories

### Requirement: Validation enums are derived from Prisma
Request body validation for `status`, `priority`, and `frequency` fields SHALL use the enum types generated by Prisma Client (`TaskStatus`, `Priority`, `HabitFrequency`) as the source of truth, instead of separately maintained literal arrays.

#### Scenario: Invalid enum value is rejected
- **WHEN** a request body includes a `status`, `priority`, or `frequency` value not present in the corresponding Prisma enum
- **THEN** the API responds with `400` and a validation error message, matching current behavior

### Requirement: TypeScript runs without a separate build step
The `dev` and `start` npm scripts SHALL run the TypeScript entrypoint directly via `tsx`, and the Vercel deployment SHALL use the TypeScript entrypoint directly, without a `tsc` compile step in the deploy pipeline.

#### Scenario: Local start runs TypeScript directly
- **WHEN** `npm start` is run locally
- **THEN** the server starts by executing `server.ts` via `tsx`, with no intermediate `dist/` output

#### Scenario: Vercel build references the TypeScript entrypoint
- **WHEN** `vercel.json` is inspected
- **THEN** its build `src` points to `server.ts`
